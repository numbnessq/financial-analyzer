# 📊 Financial Document Analyzer

Десктоп-приложение для анализа финансовых документов организации с целью выявления аномалий, потенциальных коррупционных схем и неэффективных закупок.

Система агрегирует данные из множества источников (сметы, договоры, акты), сопоставляет их между подразделениями и контрагентами, рассчитывает риск на основе вероятностной модели и визуализирует связи в виде графа.

---

# 🚀 Основные возможности

### 📂 Работа с документами

* Загрузка до **десятков документов одновременно**
* Поддержка форматов:

  * PDF
  * DOCX
  * XLSX
* Автоматический парсинг и извлечение данных

---

### 🤖 AI-обработка

* Извлечение структурированных позиций из текста (Ollama, локально)
* Нормализация названий, единиц измерения и форматов
* Устранение опечаток и транслита
* Fuzzy-matching позиций между документами

---

### 🔍 Анализ аномалий

Система выявляет:

* отклонения цен (смета vs договор)
* дублирование закупок между отделами
* размытые позиции (“прочие”, “услуги”)
* дробление закупок
* концентрацию поставщиков
* подозрительные временные паттерны
* аномалии в структуре связей (через граф)

---

### 📊 Риск-скоринг

* Вероятностная модель (не линейная)
* Учет комбинации факторов
* Итоговый риск: **0–100**
* Четкая классификация уровней риска

---

### 🧠 Объяснения

* Формируются на основе **фактов, а не генеративного текста**
* Показывают:

  * какие отделы участвуют
  * насколько отклонена цена
  * какие факторы сработали

---

### 🕸 Граф связей

* Иерархическая структура:

  ```
  Department → Item → Contractor
  ```
* Отображает:

  * связи между подразделениями
  * общие закупки
  * концентрацию поставщиков
* Подходит для визуального анализа схем

---

### 🖥 Интерфейс

* Десктоп-приложение (Electron)
* Табличный анализ рисков
* Фильтрация и сортировка
* Визуализация графа

---

# 🧠 Риск-скоринг (вероятностная модель)

Система использует формулу:

[
risk = 1 - \prod (1 - p_i)
]

где:

* ( p_i ) — вероятность отдельного фактора риска

---

## 🔍 Факторы риска

| Фактор                   | Описание                               | Вероятность |
| ------------------------ | -------------------------------------- | ----------- |
| duplicate_3_plus         | 3+ подразделений закупают одну позицию | 0.65        |
| duplicate_2              | 2 подразделения                        | 0.35        |
| vague_item               | Размытая формулировка                  | 0.70        |
| price_deviation_50       | Отклонение цены > 50%                  | 0.60        |
| price_deviation_20       | Отклонение > 20%                       | 0.40        |
| contractor_concentration | Один поставщик                         | 0.50        |
| split_suspected          | Дробление закупки                      | 0.45        |
| single_occurrence        | Недостаточно данных                    | 0.10        |
| contractor_blacklist     | Подозрительный контрагент              | 0.80        |
| temporal_clustering      | Частые закупки                         | 0.35        |
| graph_central            | Центральность в графе                  | 0.30        |

---

## 📈 Уровни риска

| Уровень     | Диапазон |
| ----------- | -------- |
| 🟢 LOW      | 0–19     |
| 🟡 MEDIUM   | 20–39    |
| 🔴 HIGH     | 40–69    |
| 🟣 CRITICAL | 70–100   |

# ⚙️ Установка и запуск

## 1. Клонирование

```bash
git clone https://github.com/numbnessq/financial-analyzer.git
cd financial-analyzer
```

---

## 2. Backend

```bash
python -m venv venv
source venv/bin/activate  # macOS/Linux
# или
venv\Scripts\activate     # Windows

pip install -r requirements.txt
```

---

## 3. Ollama

```bash
ollama pull mistral
ollama serve
```

---

## 4. Запуск backend

```bash
cd backend
uvicorn main:app --reload
```

---

## 5. Frontend

```bash
cd ..
npm install
npm start
```
